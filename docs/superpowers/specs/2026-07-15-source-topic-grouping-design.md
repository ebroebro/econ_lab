# 소스함 주제별 그룹핑 + 오늘의 추천 설계

**목표:** 소스함(`#tab-sources`) 목록을 "미국주식/국내주식/부동산정책/청약" 등 세분화된 주제 소제목으로 묶어서 보여주고, 카테고리마다 최신 소스 몇 개를 "오늘의 추천"으로 표시한다.

## 배경

현재 소스는 `type` 컬럼(`news`/`realestate`/`stock`/`manual`)만 갖고 있고, RSS 뉴스는 전부 `news`로 뭉뚱그려 저장돼 세분화된 주제 정보가 없다. 이번 기능은 DB나 수집기를 건드리지 않고, 이미 클라이언트가 받아온 제목/요약 텍스트를 **키워드 규칙으로 분류**해 화면에서만 그룹핑한다 — API 호출·수집 속도·DB 스키마에 영향 없음(사용자 확인 완료).

## 분류 방식

`src/web/public/app.js`에 순수 함수로 카테고리 규칙 테이블을 추가한다. 소스의 `title + ' ' + summary`를 우선순위 순서로 정규식과 비교해 첫 매치되는 카테고리로 분류하고, 매치가 없으면 `기타`로 둔다. 순서가 우선순위이므로 더 구체적인 카테고리(청약)를 일반적인 카테고리(부동산정책)보다 먼저 검사한다.

**카테고리 정의(순서대로 매칭):**

| 순서 | key | label | 매칭 키워드 |
|---|---|---|---|
| 1 | `subscription` | 청약 | 청약, 분양, 입주자모집, 특별공급, 당첨 |
| 2 | `realestate_policy` | 부동산정책 | 규제, DSR, 대출, 재건축, 재개발, 종부세, 재산세, 공급대책, 토론회 |
| 3 | `us_stock` | 미국주식 | 미국, 나스닥, 다우, S&P, 연준, Fed, 파월, ADR, 월가 |
| 4 | `kr_stock` | 국내주식 | 코스피, 코스닥, 삼성전자, 하이닉스, 특징주, 상한가, 실적, 증권 |
| 5 | `rate_fx` | 금리·환율 | 기준금리, 환율, 원/달러, 한은, 금통위 |
| 6 | `etc` | 기타 | (위에 안 걸리는 것 전부, 항상 매치) |

## UI 동작

- 기존 "전체 유형" 드롭다운 필터(뉴스/부동산/증시/직접입력)는 그대로 둔다 — 그 필터를 통과한 결과 안에서 다시 주제별로 묶는다.
- 목록을 카테고리 순서대로 섹션 헤더(`<h3>청약</h3>` 등) + 그 아래 기존 카드 그리드로 렌더링한다.
- 소스가 하나도 없는 카테고리는 섹션 자체를 렌더링하지 않는다.
- 서버 응답이 이미 `collected_at DESC, id DESC`로 정렬돼 오므로(변경 없음), 그룹핑 후에도 각 카테고리 내부는 최신순을 유지한다.
- 각 카테고리에서 맨 위(가장 최신) 최대 2개 소스 카드에 "⭐ 오늘의 추천" 배지를 붙인다. 카테고리에 소스가 2개 미만이면 있는 만큼만 배지를 붙인다.
- 카드 선택(클릭 → 다중 선택) 동작은 카테고리와 무관하게 기존과 동일하게 유지한다.

## 구현 범위

- `src/web/public/app.js`: 카테고리 규칙 테이블 + `classifySource(source)` 순수 함수 추가, `loadSources()`를 그룹핑 렌더링으로 수정.
- `src/web/public/style.css`: 섹션 헤더, 추천 배지 스타일 추가.
- 백엔드(`server.js`, `db.js`, collectors)는 변경하지 않는다.

## 테스트

이 저장소는 백엔드 로직(`node:test`)만 자동 테스트하고 프런트엔드 vanilla-JS는 별도 테스트 스위트가 없다(기존 네이버/티스토리 UI 작업과 동일한 패턴). `classifySource`를 `app.js`에서 분리 가능한 순수 함수로 작성해두면 향후 테스트를 붙이기 쉽지만, 이번 스코프에서는 기존 관례를 따라 브라우저 수동 확인(대시보드 실행 → 소스함에서 섹션 헤더와 추천 배지 렌더링 확인)으로 검증한다.
